1. Traditional websites: Before the rise of modern JavaScript frameworks, most websites were implemented as multi-page applications. However, this approach can be resource-intensive for web servers and may result in a slow browsing experience for users.
2. What is an SPA? A single-page application is a web app that has only one HTML page sent from the server to the browser. Instead of loading entire new web pages, an SPA updates its content dynamically as users interact with it.
3. Benefits of SPAs: SPAs provide a user-friendly, mobile-friendly experience and offer a unique, speedy, and engaging browsing experience. They allow users to interact with the website without the need to download entire new web pages, resulting in a faster and more responsive experience.
4. Serving code and resources: SPAs can use two approaches to serve code and resources. The first approach is bundling, where all necessary HTML, CSS, and JavaScript are loaded immediately. The second approach is lazy loading or code splitting, where only the minimum resources needed to load the application are initially loaded, and additional resources are downloaded as required.
5. Example scenarios: The course material provides examples to illustrate the difference between traditional websites and SPAs. For instance, in a traditional website, clicking a button may trigger a request to the server, which returns a new web page. In an SPA, the server returns a JSON object containing the necessary data, and the application updates the HTML accordingly.
6. Considerations for your website: When building a website, consider whether its complexity suits a traditional multi-page application or would perform better as an SPA. The choice of approach depends on factors such as the size, complexity, and bandwidth requirements of the application.
