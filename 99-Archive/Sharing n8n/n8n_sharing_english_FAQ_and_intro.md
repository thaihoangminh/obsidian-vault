# n8n – Quick FAQ Sheet  
**Internal Sharing – Introduction to Workflow Automation Tool**

## 1. I don’t know how to code, can I still use it?  
**Yes.** n8n has a drag-and-drop interface and most basic workflows don’t require any coding. You just need to understand the task you want to automate.

## 2. Is it free to use?  
**Yes.** n8n is open-source and can be self-hosted for free. There’s also a cloud version at [n8n.io](https://n8n.io) with free and paid plans.

## 3. Are there any simple use cases to try?  
Absolutely. For example:  
- Log form submissions into Google Sheets.  
- Send Slack or email notifications for new events.  
- Generate daily reports automatically.

## 4. Can it integrate with internal APIs?  
**Yes.** You can use the `HTTP Request` node to call any API and process data as needed.

## 5. Can it run scheduled workflows?  
**Yes.** The `Cron` node allows scheduling flows to run hourly, daily, or at custom intervals.

## 6. Is data secure? Will it be sent externally?  
**Yes.** You can self-host n8n to ensure data stays within your infrastructure. It only sends data externally if you configure it to.

## 7. Can I run multiple workflows in parallel?  
**Yes.** Each workflow runs independently and can execute simultaneously.

## 8. Does it support logging and error handling?  
**Yes.** Each execution is logged. You can also set up retry policies or send alerts when failures happen.

## 9. Why use n8n over Zapier or Make?  
- **n8n is open-source and free to self-host**, without run limits like Zapier.  
- **More flexibility**, allowing custom logic and complex workflows.

## 10. How can I learn more?  
You can:  
- Visit [docs.n8n.io](https://docs.n8n.io)  
- Watch tutorials on YouTube or the official site  
- Ask me for support with your first workflow!
